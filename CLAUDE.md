## 4. מערכת העיצוב (Design System)

מוגדרת כמשתני CSS בראש [css/style.css](css/style.css):

| משתנה | ערך | שימוש |
|---|---|---|
| `--navy` | `#0B1E3D` | צבע ראשי (כותרות, רקעים כהים) |
| `--gold` | `#C9972B` | צבע הדגשה (כפתורים, קישוטים) |
| `--off-white` | `#F8F9FB` | רקע סקשנים |
| `--gray` | `#6B7280` | טקסט גוף |
| `--radius` | `12px` | עיגול פינות |

- **גופן**: `Heebo` (Google Fonts), עם נפילה ל-Arial.
- **רספונסיבי**: breakpoint עיקרי ב-`768px`.
- מחלקות עזר: `.container` (רוחב מקס' 1200px), `.section-py`, `.btn`, `.btn-gold`, `.reveal` (אנימציית גלילה).

## 5. לוגיקת JavaScript ([js/main.js](js/main.js))

- **Navbar scroll** — מוסיף `.scrolled` בגלילה מעבר ל-60px.
- **תפריט מובייל** — כפתור המבורגר פותח/סוגר את `#mobileMenu`.
- **Scroll reveal** — `IntersectionObserver` מוסיף `.visible` לאלמנטי `.reveal`.
- **מונים מונפשים** — `.count-up` עם `data-target` ו-`data-suffix`.
- **כרטיסי צוות** — כפתור `.team-expand` פותח/מקצר טקסט.
- **פוקוס כרטיסי שירות במובייל** — מדגיש את הכרטיס הקרוב למרכז המסך.
- **קישור ניווט פעיל** — מסמן אוטומטית את העמוד הנוכחי לפי שם הקובץ.

## 6. הטפסים — שליחה אמיתית דרך FormSubmit (חינמי)

כל שלושת הטפסים (דף הבית, `contact.html`, `careers.html`) שולחים למייל **info@shk.org.il**
דרך השירות החינמי **FormSubmit** (`formsubmit.co`) — בלי backend ובלי הרשמה.

- **טופסי צור קשר** (`.contact-form`): נשלחים ב-**AJAX** מ-[js/main.js](js/main.js)
  לכתובת `https://formsubmit.co/ajax/info@shk.org.il`. נשארים בעמוד ומציגים הודעת הצלחה.
- **טופס קריירה** (`.careers-form`): שליחה **רגילה** (`action` + `enctype="multipart/form-data"`)
  כדי לתמוך ב**צירוף קובץ קו"ח** (FormSubmit תומך בקבצים עד 10MB בחינם). AJAX לא תומך בקבצים.
- שדות נסתרים בכל טופס: `_subject` (נושא המייל), `_template=table` (עיצוב), `_captcha=false`,
  ו-`_honey` (מלכודת ספאם).
- שמות השדות בעברית (שם / טלפון / אימייל / הודעה / קורות חיים) כדי שהמייל יגיע קריא.

⚠️ **הפעלה חד-פעמית**: בשליחה הראשונה FormSubmit שולח מייל הפעלה ל-info@shk.org.il —
חובה ללחוץ על קישור ה-Activate פעם אחת כדי שכל הטפסים יתחילו לעבוד.

> שדרוג עתידי: כשהאתר יהיה באוויר אפשר להוסיף `_next` עם כתובת עמוד "תודה" מותאם.

## 7. רכיב הנגישות ([js/accessibility.js](js/accessibility.js))

- תואם **תקן ישראלי 5568 / WCAG 2.0 AA**.
- נטען אוטומטית בכל דף, **בונה את כפתור ותפריט הנגישות בעצמו** (אין צורך ב-HTML).
- שומר העדפות ב-`localStorage` (מפתחות `shk_a11y`, `shk_a11y_font`).
- מצבים: ניגודיות בהירה/כהה, גווני אפור, הדגשת קישורים/כותרות, גופן קריא, ריווח טקסט,
  עצירת אנימציות, סמן גדול, הגדלת/הקטנת גופן.

## 8. הרצה מקומית

צעד אחר צעד:

1. פתח טרמינל בתיקיית האתר.
2. הרץ:
   ```
   python no_cache_server.py
   ```
3. פתח בדפדפן: `http://localhost:8000`

`no_cache_server.py` הוא שרת פייתון פשוט ששולח headers של "ללא cache" — כך כל
רענון מציג מיד את השינויים האחרונים בלי קאשינג של הדפדפן.

## 9. גיט והעלאה לגיטהאב

- **Repository**: `https://github.com/shehakim/shahakim-site` (פרטי)

### איך מעלים שינוי

**אפשרות א' (קלה) — לחיצה כפולה** על `עדכון-אתר.bat`.
מעלה הכל אוטומטית עם הודעת עדכון של תאריך+שעה. בלי שאלות.

**אפשרות ב' — פקודות ידניות:**
```
cd "c:/Users/USER/Downloads/אתר שחקים"
git add -A && git commit -m "תיאור השינוי" && git push
```

> הערה: אזהרות `LF will be replaced by CRLF` הן תקינות בווינדוס — להתעלם.

## 10. חיבור לאוויר (דומיין) — טרם בוצע

האתר **עדיין לא מחובר לאוויר** — נמצא רק בגיטהאב לפי בקשת הלקוחה.
ההמלצה כשנרצה לחבר: **GitHub Pages** (חינמי, אוטומטי בכל push).

הקבצים כבר מוכנים מראש (`CNAME` עם `tpshk.org.il`, `.nojekyll`). יישאר רק:
1. **Settings → Pages** ב-repo: Source = Deploy from a branch, Branch = `main` / root.
2. ב-internic/אינטרספייס להגדיר 4 רשומות **A** לדומיין הראשי:
   `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
3. לוודא Custom domain = `tpshk.org.il` ולסמן **Enforce HTTPS**.

## 11. כללי עריכה (חשוב!)

- **כל דף עצמאי** — שינוי בניווט/פוטר → לעדכן בכל קובצי ה-HTML.
- **שמות קבצים בעברית** (לוגו, וידאו, תמונות) — מקושרים בנתיבים יחסיים. עובדים גם
  בגיטהאב בזכות `.nojekyll`. בעת הוספת תמונה — אפשר להמשיך בעברית.
- **שדות קלט מספריים**: להשתמש ב-`placeholder` ולא ב-`value="0"`.
- אחרי כל שינוי — להעלות לגיטהאב (סעיף 9).
